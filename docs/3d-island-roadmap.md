# QuestKids 立體小島地圖 Roadmap

目標：把小任務島由平面任務地圖，逐步升級成像玩具模型世界的可探索小島。孩子感覺是「入島冒險」，不是被要求完成 checklist。

## Phase 1：2.5D Toy Island MVP

不引入 3D 引擎，先用現有 HTML/CSS/JS 做立體玩具盒感。

- 將現有地圖升級為 2.5D 分層場景：海面、島地、山路、碼頭、樹屋、家務小屋、平靜海灣。
- 每個任務區改成小型立體地標，不用大按鈕遮住背景。
- 地圖保留拖動、縮放、全景。
- 完成任務後，對應區域有可見變化：旗升起、樹發光、屋子亮燈、海灣泡泡。
- Lumo / 寵物可在路徑附近浮動，避免遮住孩子角色。
- 手機 Safari 優先，避免文字直排、橫向溢出或滾動卡死。

成功標準：

- 孩子第一眼能看出「這是一個可以探索的小島」。
- 任務區可以點、拖地圖可以看到全景。
- 不需要 WebGL，舊 iPhone 也應流暢。

## Phase 2：互動區域場景

每個區域點開後，不只是清單，而是一個小場景入口。

- 晨光碼頭：小船、旗、出發箱，對應早晨/準備任務。
- 學習樹林：樹屋、書本台、發光葉子，對應功課/閱讀。
- 家務港灣：小屋、整理箱、洗衣繩，對應家務任務。
- 平靜海灣：泡泡、月亮石、呼吸圈，對應情緒/休息。
- 點區域時先出現「第一步卡」，再進入計時器。
- 暫停時場景保持半完成，不懲罰、不清零。

成功標準：

- 孩子不用看很多文字，也知道自己在做哪一類冒險。
- 任務開始前的心理阻力下降。

## Phase 3：真 3D 小島 Prototype

用 Three.js 或等價輕量 3D 技術做可旋轉小島原型。

- 建立低多邊形 toy island：海、地形、樹、屋、碼頭、水瀑。
- 角色與 Lumo 以 sprite / billboard 方式呈現，先不做複雜骨骼動畫。
- 支援旋轉、縮放、點選區域。
- WebGL 不可用時 fallback 回 Phase 1 的 2.5D 地圖。
- 只在孩子端地圖使用，不影響家長端。

成功標準：

- iPhone Safari 可用，畫面不空白。
- 3D 不阻礙任務流程。
- fallback 穩定。

## Phase 4：商業版建造與長期留存

把完成任務和收藏轉化為「建造自己的島」。

- 任務獎勵可解鎖島上裝飾：樹、燈塔、橋、寵物屋、收藏館。
- 卡牌可放進圖鑑館 / 展示台。
- 生活能力任務解鎖實用建築，例如繩結橋、摺紙船、鞋帶跑道。
- 每週故事事件帶來新裝飾或小區域。
- 家長端可以設定獎勵上限，避免孩子為獎勵過度刷任務。

成功標準：

- 長期回訪理由從「做任務」變成「照顧和建造我的小島」。
- 商業化 iPhone 版本有可持續擴展的收藏與世界觀。

## 實作原則

- 先 2.5D，再 3D。不要一開始跳進 Roblox 級自由世界。
- 任務永遠是主流程，地圖是降低抗拒感的入口。
- 所有失敗、暫停、放棄狀態都保留進度，不做懲罰。
- 小朋友 UI 優先用圖像、角色、位置感，不用長文字。
- 家長端保持清楚、克制，不做遊戲化干擾。
