# CHANGELOG

## 2026-06-25

### 今日陪伴隊

- 孩子小島首頁新增「今日陪伴隊」，把已收集的寵物夥伴帶回任務主流程。
- Lumo 會按今日第一步或已完成狀態說一句短對白，並直接顯示下一步要做什麼。
- 已擁有的寵物卡會挑選最多 3 個出場，每個夥伴都有自己的短句，讓收藏不只停留在圖鑑。
- 新增「和夥伴開始」與「看看夥伴」入口，讓孩子可以由角色陪伴自然進入任務或圖鑑。
- 手機版加入單欄與縮小寵物卡處理，避免新面板擠壓任務清單。
- 同步更新 `index.html` 的 `styles.css` / `app.js` cache query。

### Lumo 和寵物生命感

- 新增共用角色反應系統，Lumo 會按「開始、猶豫、暫停、完成、開卡包」切換表情、微動畫和一句短對白。
- 任務卡住選擇頁、第一步魔法卡、暫停鼓勵、任務完成和卡包畫面都加入 Lumo 的情境反應，讓孩子感覺是被陪伴而不是被催促。
- 抽到寵物卡時，卡牌揭曉會顯示該寵物自己的短對白，例如 Sprout、Mochi、泡泡龜、緊張獸等。
- 浮動旁白收起後的 icon 也改用 Lumo 表情，打開後會跟目前任務狀態同步。
- 卡包視窗改為可縱向捲動，避免新增角色內容後在手機或 Safari 被截斷。
- 同步更新 `index.html` 的 `styles.css` / `app.js` cache query。

### Safari 版面修正

- 修正 Safari / iPad 尺寸下「今日島嶼故事」文字被壓成直排、故事卡高度爆開的問題。
- 將故事卡改為更保守的兩欄布局，進度卡移到底部全寬顯示，避免中文標題被第三欄擠窄。
- 明確設定孩子端與故事卡使用橫向書寫和正常換行，防止 Safari 將中文逐字排版。
- 收緊橫向專注模式觸發條件，只在真正低高度手機橫向時隱藏普通內容。
- 同步更新 `index.html` 的 cache query，讓 Safari 重新載入新版 CSS。

### 每日島嶼故事

- 孩子端小任務島新增「今日島嶼故事」入口，每天可先選一條故事線：尋找晨光貝殼、種下發光種子、點亮海灣燈塔。
- 選擇故事線後，任務會推進故事里程碑：選定冒險、完成第一個小任務、集齊卡包線索。
- 故事面板加入 Lumo 旁白、場景圖示、進度條和「推進故事」主操作，讓孩子有開始任務的想像理由。
- 故事選擇按孩子及日期保存，不會改動家長任務庫，也不會把其他任務隱藏。
- 介面新增手機版單欄布局，避免故事卡和文字在小螢幕擠在一起。
- 同步更新 `index.html` 的 `styles.css` / `app.js` cache query。

### 卡包羅盤與透明保底

- 新增按孩子保存的卡包紀錄（`state.packLedger`），追蹤總開包數、距離上次能力卡、區域線索及最近發現。
- 卡包選卡改為透明規則：未收齊前優先新收藏，能力卡最多 10 包內會出現，最近任務區域最多 5 包內會優先出現相關卡。
- 孩子小島、圖鑑和開包畫面加入「卡包羅盤」，顯示新收藏、能力卡和區域線索，不顯示機率或刺激式稀有倒數。
- 卡牌揭曉畫面加入本次出卡原因，例如「能力卡保底」或「學習樹林線索」，讓家長理解不是黑箱抽卡。
- 「探索星」開包成本改用同一個規則常數，避免 UI 文案和實際邏輯日後不一致。
- 圖鑑頁新增獨立的卡包規則面板，孩子可以看到下一包的安全預期和最近發現。
- 同步更新 `index.html` 的 `styles.css` / `app.js` cache query，方便重新整理。

## 2026-06-23

### 每日獎勵上限（第一階段）

- 新增按孩子 + 香港日期保存的每日獎勵紀錄（`state.dailyRewards`），包含 `xpEarned` 與 `packsOpened`，舊 localStorage 自動補齊，不會清除現有資料。
- 每日 XP 上限 60：所有任務完成（包括能力任務確認）都經 `applyDailyXpAward()` 走同一個 helper，超出時只截斷今日 XP 增量，累積 XP 仍保留。
- 每日卡包上限 2：第 3 包起按鈕停用並顯示「今天已探索好多，明天再開新一包」，不扣探索星、不新增收藏、不顯示失敗。
- 孩子小島及圖鑑卡包區同步顯示「今日已開 X/2 包」與「今天還可以開 N 包」透明提示；達上限後改為鼓勵明日再試。
- 完成任務慶祝畫面改為顯示本次實際獲得 XP；若觸發上限或為 0 XP，會以溫和文字「今天的 XP 已收集完成」說明。
- 家長總覽新增「今日 XP 入帳 X/60」與「今日已開 X/2 包」指標，但不新增額外設定 UI。
- 達上限時會顯示一次性小提示氣泡（3.2 秒後自動消失），不擋住操作也不會嚇到孩子。
- 跨日自動使用新紀錄，沿用 `todayKey()` 的 Asia/Hong_Kong 邏輯，無需手動清理舊紀錄。
- 同步更新 `index.html` 的 `styles.css` / `app.js` cache query，方便重新整理。

## 2026-06-22

### 每週任務排程

- 每項任務新增「出現星期」設定（星期日至六），家長可逐個開關；預設為每天顯示。
- 家長任務管理每列加入七個簡潔的星期按鈕，並顯示「每天／平日／週末／自訂星期」摘要。
- 舊資料沒有 `weekdays` 時會自動遷移為每天，孩子與家長都不會看到今日清單突然消失。
- 孩子端、地圖區域數量、必須任務進度、第一步魔法卡及家長今日焦點，只計算今天有排程的任務。
- 「今天顯示／暫停安排」仍是總開關，暫停後任何星期都不顯示。
- 每日排序只保存當天有排程的任務 ID，下一天切換時不會被其他星期的紀錄污染。
- 響應式：390px 手機版星期按鈕自動收窄，不會橫向溢出。

### 孩子任務拖動排序

- 在「自己排次序」模式下加入拖動把手，孩子可用滑鼠或觸控直接拖動任務卡重新排次序。
- 拖動期間會顯示抬起、佔位及目標位置的視覺狀態，觸控時不會令頁面意外捲動。
- 拖動落點改為按其他任務卡的中心位置計算，避免清單標題高度令卡片落錯位置。
- 移動與放手事件由視窗層接收，手指離開細小拖動把手後仍能可靠完成排序。
- 拖動把手以外的文字、箭嘴及任何按鈕都不會誤觸拖動，並保留原有上下箭嘴供鍵盤使用。
- 新排序同樣按孩子及日期寫入 `dailyTaskOrders`，必須與自選任務都保留在今日清單內。

### 孩子每日自主安排

- 家長端可把任務標示為「必須完成」或「自選任務」。
- 孩子端加入「自己排次序」，可用箭嘴自行決定今日任務先後。
- 孩子的排序按日期及人物獨立保存，不會改亂家長的任務庫。
- 小島進度改為反映必須任務完成率，完成全部核心項目才達成今日目標。
- 新增任務時可直接決定是否列為今日必須項目。

### 地圖人物臉部

- 為地圖上的孩子角色加入獨立臉部比例，避免縮小後雙眼連成遮擋。
- 調整頭髮、臉部、衣領的圖層次序，讓表情保持清楚。
- 將 Lumo 移離孩子臉部及身體，兩個角色不再互相重疊。

### 地圖縮放顯示

- 縮放按鈕會即時顯示目前實際倍率。
- 地圖縮放會以 0.1x 為一級，支援 0.5x 至 1.2x。
- 到達最大或最小倍率時停用相應按鈕，重設按鈕會回到 1x。
- 地圖改用完整原圖比例，可上下左右拖動查看所有區域。
- 加入全景按鈕，一按以 0.5x 顯示整座島嶼。

### 家長今日焦點

- 重製家長總覽首屏，集中顯示孩子今天狀態、完成率及下一個任務。
- 根據心情與任務進度生成可直接使用的陪伴說話。
- 加入「陪他開始」入口，家長可由總覽直接帶孩子進入下一任務。
- 將能力任務與收藏數量收納到今日焦點，減少資訊分散。

### 親子心聲互動

- 孩子端以家長頭像和心聲卡顯示最新訊息，可標記收好。

### 每日任務編排

- 家長可用上移及下移按鈕調整孩子端的任務順序。
- 每項任務可暫停或重新加入今日安排，保留原有紀錄及內容。
- 孩子端任務、第一步魔法卡及完成率會跟隨最新編排。

## 2026-06-12

按照「QuestKids 修改工作單 v1（Codex 執行版・含完整 code）」完成以下修正。

### Fix 1: 輸入內容轉義

- 新增 `esc()`，統一處理 HTML 字元轉義。
- 把孩子名稱、任務名稱、區域名稱、能力任務名稱、獎勵名稱、家長話術與步驟文字等使用者或可變內容，改為先經過 `esc()` 再輸出。

### Fix 2: 計時器局部更新

- 計時器畫面加入 `data-timer-ring` 與 `data-timer-display`。
- 新增 `updateTimerDisplay()`，倒數期間只更新計時器文字與圓環，不再每秒整頁重繪。
- `startTimer()` 在倒數結束前只做局部更新，結束時才保存狀態並重繪。

### Fix 3: 狀態遷移

- 新增 `STORE_VERSION = 5`。
- `loadState()` 改為透過 `migrateState()` 載入舊資料。
- 遷移時補齊缺少欄位、校正 `selectedChildId`，並把舊計時器狀態重設為安全初始值。
- 保持 `STORE_KEY = "questkids.mvp.v4"` 不變。

### Fix 4: 完成動作加確認閘

- 原本直接完成任務的邏輯改名為 `finalizeTask()`。
- 原本直接完成能力任務的邏輯改名為 `finalizeSkillMission()`。
- `completeTask()` 改為先檢查是否未做步驟且計時不足 30 秒，必要時先顯示確認視窗。
- `completeSkillMission()` 改為先顯示完成練習確認，再真正解鎖能力。
- `runConfirmedAction()` 改為根據 `pendingConfirm.action` 執行對應後續動作。

### Fix 5: 文案統一

- 將本次修正範圍內的介面文案統一為書面中文。
- 清理 `app.js` 內指定的粵語口語字詞，避免同頁混用。
