# QuestKids MiniMax Code 工作單

## 協作規則

- MiniMax Code 負責實作與本地測試。
- Codex 負責規劃、審查、瀏覽器驗收、commit、push 與發布。
- 只修改這個 repository 內與工作單直接相關的檔案。
- 不要執行 `git commit`、`git push`、重置或還原現有改動。
- 保留 Traditional Chinese UI，以及現有靜態 HTML/CSS/JavaScript 架構。
- 完成後列出修改檔案、測試結果、已知限制，然後停止。

## 工作單 QK-001：孩子拖動排列每日任務

目前孩子端已有「自己排次序」模式，使用上下箭嘴調整每日任務順序。

請加入真正的拖動排序：

1. 桌面可用滑鼠拖動任務卡。
2. 手機可用觸控拖動，拖動時不要令頁面意外捲動。
3. 顯示清楚的拖動把手，不要要求孩子拖任務文字或開始按鈕。
4. 拖動期間要有抬起、佔位及目標位置的視覺狀態。
5. 放開後沿用現有 `dailyTaskOrders` 按孩子及日期保存。
6. 保留上下箭嘴，供鍵盤操作及拖動失敗時使用。
7. 必須任務與自選任務都可以重新排序，但不能從今日清單移除。
8. 不要改動家長端的必須／自選設定邏輯。

## 驗收條件

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 390px 手機寬度沒有橫向溢出。
- 滑鼠及觸控拖動均能改變第一步魔法卡的下一任務。
- 點擊任務開始按鈕不會誤觸拖動。
- 現有箭嘴排序仍可使用。

## 工作單 QK-002：每週任務排程與今日清單

QK-001 完成後直接繼續本工作單，不需要等待確認。

家長需要為每項任務設定星期幾出現，孩子每天只看到當日安排。

1. 每項任務新增 `weekdays` 設定，使用 `0-6` 表示星期日至星期六。
2. 舊資料沒有 `weekdays` 時，遷移為每天顯示。
3. 家長任務管理加入七個簡潔的星期按鈕，可多選。
4. 新增任務表單預設為每天。
5. 孩子端、地圖區域數量、必須任務進度和第一步魔法卡，只計算今天有排程的任務。
6. 「今天顯示／暫停安排」仍是總開關；停用後任何星期都不顯示。
7. 每日排序只保存當天有出現的任務，不應因其他星期的任務而損壞。
8. 家長端清楚顯示「每天」、「平日」、「週末」或自訂星期摘要。
9. 不要加入日期選擇器、通知或雲端功能。

## QK-002 驗收條件

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 舊 localStorage 資料可正常載入。
- 切換星期設定後，孩子清單和地圖統計同步改變。
- 390px 手機版星期選擇不會橫向溢出。
- 不要執行 `git commit` 或 `git push`。
