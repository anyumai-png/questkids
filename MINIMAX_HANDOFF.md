# QuestKids MiniMax Code 工作單

## 協作規則

- MiniMax Code 負責實作與本地測試。
- Codex 負責規劃、審查、瀏覽器驗收、commit、push 與發布。
- 只修改這個 repository 內與工作單直接相關的檔案。
- 不要執行 `git commit`、`git push`、重置或還原現有改動。
- 保留 Traditional Chinese UI，以及現有靜態 HTML/CSS/JavaScript 架構。
- 完成後列出修改檔案、測試結果、已知限制，然後停止。

## 工作單 QK-001：孩子拖動排列每日任務

目前孩子端已有「自己排次序」模式，使用上下箭嘴調整每日任務順序。

請加入真正的拖動排序：

1. 桌面可用滑鼠拖動任務卡。
2. 手機可用觸控拖動，拖動時不要令頁面意外捲動。
3. 顯示清楚的拖動把手，不要要求孩子拖任務文字或開始按鈕。
4. 拖動期間要有抬起、佔位及目標位置的視覺狀態。
5. 放開後沿用現有 `dailyTaskOrders` 按孩子及日期保存。
6. 保留上下箭嘴，供鍵盤操作及拖動失敗時使用。
7. 必須任務與自選任務都可以重新排序，但不能從今日清單移除。
8. 不要改動家長端的必須／自選設定邏輯。

## 驗收條件

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 390px 手機寬度沒有橫向溢出。
- 滑鼠及觸控拖動均能改變第一步魔法卡的下一任務。
- 點擊任務開始按鈕不會誤觸拖動。
- 現有箭嘴排序仍可使用。

## 工作單 QK-002：每週任務排程與今日清單

QK-001 完成後直接繼續本工作單，不需要等待確認。

家長需要為每項任務設定星期幾出現，孩子每天只看到當日安排。

1. 每項任務新增 `weekdays` 設定，使用 `0-6` 表示星期日至星期六。
2. 舊資料沒有 `weekdays` 時，遷移為每天顯示。
3. 家長任務管理加入七個簡潔的星期按鈕，可多選。
4. 新增任務表單預設為每天。
5. 孩子端、地圖區域數量、必須任務進度和第一步魔法卡，只計算今天有排程的任務。
6. 「今天顯示／暫停安排」仍是總開關；停用後任何星期都不顯示。
7. 每日排序只保存當天有出現的任務，不應因其他星期的任務而損壞。
8. 家長端清楚顯示「每天」、「平日」、「週末」或自訂星期摘要。
9. 不要加入日期選擇器、通知或雲端功能。

## QK-002 驗收條件

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 舊 localStorage 資料可正常載入。
- 切換星期設定後，孩子清單和地圖統計同步改變。
- 390px 手機版星期選擇不會橫向溢出。
- 不要執行 `git commit` 或 `git push`。

## 工作單 QK-3D-001：Phase 1 2.5D Toy Island MVP

目標：把現有平面地圖升級成「立體玩具小島」感，但不引入 Three.js / WebGL。

背景文件：`docs/3d-island-roadmap.md`

實作範圍：

1. 主要修改 `app.js` 的 `islandMap()` 和 `styles.css` 的 `.map-*` / `.zone-*` 樣式。
2. 保留現有地圖拖動、縮放、全景和點選區域功能。
3. 四個區域要更像小地標，不要用大文字卡遮住背景：
   - 晨光碼頭：碼頭 / 小船 / 旗
   - 學習樹林：樹屋 / 書本 / 發光葉
   - 家務港灣：小屋 / 整理箱 / 洗衣繩
   - 平靜海灣：水池 / 泡泡 / 月亮石
4. 進度要反映在地標上，例如完成後亮燈、升旗、發光、泡泡變多。
5. Lumo 和孩子角色位置要分開，不能遮住孩子的臉。
6. 手機 Safari 390px 寬度必須正常，避免直排文字和橫向溢出。
7. 更新 `CHANGELOG.md` 和 `index.html` cache query。

驗收條件：

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 地圖可上下左右拖動和縮放。
- 點四個地標仍可切換任務區。
- 按鈕/地標沒有大面積遮住背景圖。
- 390px 手機寬度沒有橫向溢出。
- 不要執行 `git commit` 或 `git push`。

## 工作單 QK-3D-002：Phase 2 互動區域場景

QK-3D-001 完成後繼續。

目標：每個區域點開後，任務不只是清單，而是有區域場景和第一步引導。

實作範圍：

5. 暫停/未完成不扣分，不顯示失敗語氣。
6. 更新 `CHANGELOG.md` 和 `index.html` cache query。

驗收條件：

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 點每個區域後看到不同場景。
- 已完成區域有正向完成狀態。
- 手機版場景卡不遮住底部 tab bar。

## 工作單 QK-3D-003：Phase 3 真 3D 小島 Prototype

QK-3D-002 完成並經 Codex 驗收後才做。

目標：建立可選擇啟用的真 3D 小島 prototype。必須有 fallback。

限制：

- 可以引入 Three.js，但只能用在孩子端地圖。
- WebGL 不可用或低效能時，必須回到 Phase 1 的 2.5D 地圖。
- 不要重寫整個 app。

實作範圍：

1. 新增獨立 3D renderer 模組或清楚分隔的 JS 區塊。
2. 低多邊形小島：海、地形、樹、屋、碼頭、海灣。
3. 支援旋轉、縮放、點選四個區域。
4. 角色 / Lumo 可以先用 sprite 或簡單 billboard。
5. 3D 開關應可由 state 或 feature flag 控制。
6. 更新 `CHANGELOG.md` 和 `index.html` cache query。

驗收條件：

- `node --check app.js` 通過。
- `git diff --check` 通過。
- WebGL 模式不空白。
- fallback 模式可用。
- 不影響家長端。

## 工作單 QK-3D-004：Phase 4 建造與長期留存

QK-3D-003 完成後才做。

目標：讓孩子用完成任務得到的收藏逐步裝飾小島，形成長期回訪理由。

實作範圍：

1. 任務完成 / 卡牌解鎖可以帶來小島裝飾。
2. 裝飾可出現在 2.5D 或 3D 地圖上。
3. 圖鑑卡可連接到收藏館 / 展示台概念。
4. 生活能力卡解鎖對應地圖元素，例如繩結橋、摺紙船、鞋帶跑道。
5. 家長端要能看到裝飾/獎勵設定，不讓孩子為獎勵過度刷任務。
6. 更新 `CHANGELOG.md` 和 `index.html` cache query。

驗收條件：

- `node --check app.js` 通過。
- `git diff --check` 通過。
- 至少 4 種裝飾可由現有資料狀態驅動。
- 不增加賭博式稀有度刺激。
- 商業版方向清楚，但 MVP 仍保持簡潔。
