# QuestKids 獎勵經濟表（WO-003）

> 適用範圍：QuestKids「小任務島」MVP（5–10 歲兒童、家長陪伴、純本地儲存）。
> 目標：建立一個**可持續、透明、沒有賭博感**的獎勵經濟，讓完成日常任務自然推進收藏與島嶼成長。
> 本文件只定義產品規格，**不修改 `app.js` / `styles.css` / `index.html`**；附錄提供可轉成 JavaScript constants 的資料欄位，供下一個實作單使用。

---

## 0. 設計原則（Guardrails）

1. **不引入付費貨幣**。所有可消耗資源（探索星、XP、成長糖果）只能透過完成任務取得。
2. **沒有付費抽卡、loot box、隨機稀有度**。卡包只決定「從哪張卡開始介紹」，不會用低機率刺激重複抽。
3. **沒有失敗扣分**。任何狀態都不會讓已得的 XP / 探索星 / 收藏減少。
4. **沒有 streak 懲罰**。連續中斷只會凍結連續日數顯示，不會扣 XP、不會鎖住功能。
5. **沒有稀有度閃爍誘惑**。卡面稀有度只影響插畫與邊框（已實作 `rarity`），不影響中獎率、不會倒數。
6. **沒有社交排名 / 同儕比較**。
7. **能力卡必須由真實技能任務解鎖**。抽到只是登記練習，不會立即成為「已學會」。
8. **家長可調、兒童不能調**：所有公式都允許家長在「安全範圍」內覆寫，但絕不允許孩子關閉獎勵或修改上限。
9. **數值前後一致**：本文件所有數值互相對齊，下列三個例子都能逐表計算得到相同結果。

## 1. 資源（Resources）

| 名稱 | 用途 | 取得方式 | 上限／節流 |
|---|---|---|---|
| XP（探索分數） | 個人進度、兌換家長獎勵 | 完成任務（見 §2） | 每日 60 XP 上限（避免通脹） |
| 探索星 ⭐ | 開卡包 | 完成任務（每完成 1 個任務 +1 顆） | 集滿 3 顆即可開 1 包卡，多餘的保留 |
| 成長糖果 🍬 | 重複卡的升級素材 | 抽到重複卡 +1 顆 | 僅由重複卡產生，不直接給 |
| 連續日數 🔥 | 鼓勵性質的顯示值 | 連續 1 天 ≥ 1 個任務完成 | 中斷後歸零，**不影響其他資源** |
| 卡包開啟次數 | 控制抽卡節奏 | 3 顆探索星換 1 包 | 每日最多開 2 包（避免日卡堆） |
| 徽章 🏅 | 一次性成就 | 完成里程碑條件 | 不可重複獲得，已得鎖定 |

> **設計理由**：XP 與探索星並行，是因為 XP 屬於「持續累積的內在動力」、探索星屬於「短週期可兌換的具體獎勵」；分開能避免刷分疲勞，並把卡包節奏拆成可預測的「3 個任務 1 包」。

## 2. 任務分值（Task Rewards）

> 對齊現有程式：`task.xp`（任務 XP）、任務完成時額外 +1 探索星、`finalizeSkillMission` 時 +25 XP / +1 探索星。

### 2.1 基礎分值表

| 任務類型 | 預設分鐘 | 基礎 XP | 觸發區域（zone） | 範例 |
|---|---:|---:|---|---|
| **必須任務（Required）** | 5–25 分鐘 | 25 XP | 對應家長設定的 area | 準備書包、做功課、整理房間 |
| **自選任務（Optional）** | 5–15 分鐘 | 15 XP | 對應家長設定的 area | 睡前放鬆、幫忙擺碗筷 |
| **技能任務（Skill Mission）** | 3–8 分鐘 | **完成整個練習 +25 XP** | 從能力卡 `skillMission` 帶出 | 綁鞋帶、摺紙船、晨光伸展 |

### 2.2 連續完成加成（Streak Bonus，封頂）

> **加成只加 XP，不影響探索星或卡包節奏**。中斷連續不會倒扣。

| 連續完成同一區域任務天數 | 加成係數（XP） |
|---|---:|
| 1–2 天 | ×1.00 |
| 3–6 天 | ×1.10 |
| 7–13 天 | ×1.20 |
| 14–29 天 | ×1.30 |
| 30 天以上 | ×1.35（封頂，不再疊加） |

公式：`bonusXP = baseXP × streakMultiplier − baseXP`（只加差額；探索星不變，仍 +1）

### 2.3 時間加成（Effort Bonus）

| 完成時專注秒數（timer 經過秒數） | 加成係數 |
|---|---:|
| 0–29 秒 | ×0.5（仍給予，但視為「未真正開始」） |
| 30–299 秒 | ×1.00（標準） |
| 300–899 秒 | ×1.10 |
| 900 秒以上 | ×1.15（封頂，避免無聊刷時數） |

> 對齊現有 `completeTask`：未計時或 < 30 秒會跳確認框。

### 2.4 難度加成（Difficulty Bonus）

由家長新增任務時下拉選擇；兒童端看不到這個分數。

| 難度 | 條件 | 加成係數 |
|---|---|---:|
| 入門（default） | 一般任務 | ×1.00 |
| 挑戰 | 超過 20 分鐘或包含 3 步以上 | ×1.15 |
| 勇敢一試 | 從未做過的區域 | ×1.20（一次性，之後回到入門） |

> 三種加成**不疊加**：取三者中最高的一個，避免數值通脹。

### 2.5 完成一個任務的實際獎勵（彙整公式）

```
earnedXP = clamp(baseXP × max(streakMultiplier, effortMultiplier, difficultyMultiplier),
                 min=baseXP × 0.5, max=baseXP × 1.35)
earnedStars = 1  // 永遠 +1 探索星；不因加成改變
```

| 保底規則 | 數值 | 觸發後行為 |

## 4. 卡池與區域掛鉤（Zone-Linked Pool）

> 對齊 `zoneCatalog` 與 `cardCatalog.zone`。每張卡只掛一個區域，孩子完成該區域任務會「稍微提高」那張卡在卡包輪轉裡的出現順位。

### 4.1 區域掛鉤表

| 區域 | 區域 ID | 包含系列 | 任務關鍵詞 |
|---|---|---|---|
| 晨光碼頭 | `morning` | 食物、寵物（Skipper）、物件（整理背包）、能力（綁鞋帶、晨光伸展）、名勝（馬丘比丘） | 出門、書包、早餐、上學 |
| 學習樹林 | `study` | 物件（專心小燈、葉子指南針、勇氣貼紙）、能力（綁鞋帶）、名勝（艾菲爾鐵塔、吉薩金字塔）、恐龍卡 | 功課、默書、閱讀、學校 |
| 家務港灣 | `home` | 寵物（Pebble、Kiko）、能力（毛巾摺疊、繩結）、名勝（萬里長城） | 房間、收拾、玩具、衣物 |
| 平靜海灣 | `calm` | 寵物（Lumo、Mochi、泡泡龜、緊張獸、Mimo）、能力（摺紙船、泡泡呼吸）、名勝（富士山、大堡礁）、裝飾 | 睡前、放鬆、呼吸、休息 |

### 4.2 卡牌類型權重（透明輪轉表，非機率）

| 卡牌類型 | 系列標記 | 預設權重 | 對應家長的可調項目 |
|---|---|---:|---|
| 情緒小夥伴（pet） | ✦ | 30% | 不可調 |
| 動物圖鑑（item） | ◆ | 25% | 不可調 |
| 能力卡（skill） | ★ | **每 10 包必出 1 張**（見 §3.2 保底） | 家長可關閉「能力卡」輪轉，關閉後由裝飾補位 |
| 名勝（place） | ⌂ | 15% | 不可調 |
| 美食（food） | ● | 5% | 不可調 |
| 地圖裝飾（deco） | ❖ | 5% | 不可調 |
| （共） |  | 100% |  |

> 權重**只影響輪轉次序**，不影響玩家體感機率。孩子端只看到「下一包可能遇見」3 張，全部類型在 10 包內都會出現至少一次（由 §3.2 保底保證）。

### 4.3 區域本週主打

- 家長可在每週日選擇一至兩個區域作為「本週主打」。
- 該區域的卡在輪轉表中**排序提前**，但不會出現「只抽主打區域」的情況。
- 孩子端看不到「主打」字眼，只會在「下一包可能遇見」看到更多該區域卡。

## 5. 進度曲線（7 日 / 30 日 / 90 日）

> **假設**：5–10 歲孩子在典型日常節奏下完成 2–4 個任務／天。下列數值皆以「日均 3 個任務、90% 為必須任務」為基準。

|---:|---:|---:|---:|---:|---:|
| Day 1 | 2 個 | 50 XP | 2 | 0 | 0 |
| Day 2 | 3 個 | 60 XP（原始 75，套用上限） | 3 | 1 | 1 新卡 |
| Day 3 | 3 個 | 60 XP（原始 80，套用上限） | 3 | 1 | 1 新卡 |
| Day 4 | 3 個 | 60 XP（原始 80，套用上限） | 3 | 1 | 1 新卡 |
| Day 5 | 4 個 | 60 XP（原始 110，套用上限） | 4 | 1 | 1 新卡 |
| Day 6 | 3 個 | 60 XP（原始 85，套用上限） | 3 | 1 | 0–1 |
| Day 7 | 3 個 | 60 XP（原始 95，套用上限） | 3 | 1 | 1 技能卡或名勝 |
| **7 日合計** | **21 個任務** | **410 XP** | **21 顆（最多 6 包卡）** | **6 包** | **約 5 張新卡** |

### 5.2 30 日穩定期

| 指標 | 數值 | 備註 |
|---|---:|---|
| 日均完成任務 | 3 個 | 對齊示範任務預設 |
| 日均 XP | 55–60 XP（套用上限後） | 連續加成可令原始 XP 增加，但每日入帳最多 60 |

| 月徽章新增 | 1–2 枚 | 對齊 `badgeCatalog` |

### 5.3 90 日長期曲線

| 階段 | 日期 | 重點 | XP 累積估計 |
|---|---|---|---:|
| 新手 | Day 1–7 | 熟悉島嶼、得到首批 5 張收藏 | ~410 XP |
| 養成 | Day 8–30 | 探索星穩定收穫、開始集徽章 | ~1,750 XP |
| 穩定 | Day 31–60 | 大部分可收集卡解鎖、剩餘重複卡 | ~3,500 XP |
| 收成 | Day 61–90 | 重複卡 → 等級 + 糖果；家長可考慮新增任務類型 | ~5,250 XP |

### 5.4 防通脹設計

| 上限類型 | 數值 | 說明 |
|---|---:|---|
| 每日 XP 獲得 | **60 XP** | 超出不計入，避免一天內刷高 |
| 每日開包 | **2 包** | 第 3 包開始提示「明天再來」 |
| 連續加成 | **×1.35** | 30 天後不再上漲 |
| 重複卡 XP | **0** | 重複不給 XP |
| 任務難度加成 | **×1.20 一次性** | 鼓勵探索，不會疊加 |

## 6. 反沉迷與兒童安全限制（Safety Limits）

| 項目 | 限制 | 說明 |
|---|---|---|
| 付費抽卡 | ❌ 禁用 | 無任何真實或模擬付費入口 |
| 限時卡池 | ❌ 禁用 | 卡池全公開、無倒數、無「限定」字眼 |
| 稀有度機率 | ❌ 禁用 | 卡牌稀有度只影響外觀，不影響中獎率 |
| 失敗扣分 | ❌ 禁用 | 已得 XP／星／糖果不會減少 |
| Streak 懲罰 | ❌ 禁用 | 中斷只凍結顯示，不影響獎勵 |
| 排行榜 | ❌ 禁用 | 沒有任何同儕比較 |
| 廣告 | ❌ 禁用 | 孩子端不嵌入廣告 |
| 通知推播 | ⚠️ 預設關閉 | 家長可在「家長端 → 安全私隱」打開 |
| 計時器上限 | **25 分鐘** | `timerSecondsForAge` 已限制在 10–25 分鐘範圍 |
| 每日裝置停留 | ⚠️ 將於 v2 引入 | 先記錄於本機，不作任何限制 |
| 陌生人互動 | ❌ 禁用 | 沒有聊天、加好友、分享按鈕 |

## 7. 三個玩家例子（可逐表計算）

### 例子 A：7 歲「輕量日」（小晴，Day 2）

- 角色：5–7 歲，預設專注 15 分鐘（`ageMinutes(7)=15`）。
- 當日完成：1 個必須任務（準備書包，10 分鐘）、1 個自選任務（睡前放鬆，5 分鐘）。
- 連續：2 天（無加成，仍 ×1.00）。
- 難度：入門（×1.00）。
- 時間：準備書包 600 秒 → ×1.10；睡前放鬆 300 秒 → ×1.10。

| 任務 | base | 最高加成 | 計算 | earned XP |
|---|---:|---|---|---:|
| 準備書包（必須） | 25 | ×1.10（時間） | 25 × 1.10 | 27.5 → 27 XP |
| 睡前放鬆（自選） | 15 | ×1.10（時間） | 15 × 1.10 | 16.5 → 16 XP |
| **當日 XP** |  |  |  | **43 XP**（< 60 上限） |
| 探索星 |  |  |  | **2 顆** |
| 可開卡包 |  |  |  | **0 包** |
| 連續日數 |  |  |  | **2 天** |

驗算：與 §5.1 Day 2（3 個任務 75 XP / 3 顆星）相比，少 1 個任務即少約 25 XP 與 1 顆星，**符合曲線**。

### 例子 B：9 歲「正常日」（小明，Day 14）

- 角色：8–10 歲，預設專注 20 分鐘。
- 當日完成：3 個必須任務（做功課 20 分、整理房間 15 分、幫忙擺碗筷 10 分）。
- 連續：14 天（×1.30）。
- 難度：入門 ×1.00；挑戰 ×1.15（整理房間由家長標示為挑戰）。
- 時間：均 ≥ 300 秒，×1.10。

| 任務 | base | 最高加成 | 計算 | earned XP |
|---|---:|---|---|---:|
| 做功課（必須、入門） | 25 | ×1.30（連續） | 25 × 1.30 | 32.5 → 32 XP |
| 整理房間（必須、挑戰） | 25 | ×1.30（連續 > 挑戰 ×1.15） | 25 × 1.30 | 32.5 → 32 XP |
| 幫忙擺碗筷（自選） | 15 | ×1.30 | 15 × 1.30 | 19.5 → 19 XP |
| **當日 XP** |  |  |  | **83 XP** |
| 套用每日上限 |  |  |  | **clamp 至 60 XP** |
| 探索星 |  |  |  | **3 顆** |
| 可開卡包 |  |  |  | **1 包** |
| 連續日數 |  |  |  | **14 天** |

驗算：對齊 §5.1 Day 7「×1.20 → 原始 95 XP，再套用 60 XP 上限」；本例乘 1.30 同樣觸發 60 XP 上限，**符合曲線**。

### 例子 C：「技能任務日」（小晴，抽到「綁鞋帶」能力卡）

- 背景：Day 5 抽到 `skill_shoelace`，自動建立技能任務 `skill_shoelace`。
- 孩子完成 3 個小步驟：把兩條鞋帶交叉拉緊、做一個小圈圈、另一條鞋帶繞圈再穿過洞（共 5 分鐘，300 秒）。
- 家長按「解鎖能力」→ `finalizeSkillMission`。

| 項目 | 計算 | 結果 |
|---|---|---:|
| 練習完成 XP（`finalizeSkillMission`） | 固定 +25 XP | 25 XP |
| 練習完成探索星 | 固定 +1 顆 | 1 顆 |
| 時間加成 | 300 秒 → ×1.10 | 25 × 1.10 = 27.5 |
| 連續加成 | 5 天 ×1.10 | 不疊加，取較高 ×1.10 |
| 最終 XP | clamp 至 base×1.35 = 25 × 1.35 = 33.75 | **27 XP**（採時間加成，較高者；尚未到上限 60） |
| 探索星 | | **+1 顆** |
| 連續日數 | | 沿用當日既有連續，不變動 |

驗算：與 §2.5 公式 `earnedXP = base × max(multipliers)` 一致；技能任務只加 1 顆星，避免打斷日常節奏。

## 8. 可轉成 JavaScript Constants 的資料表（供下一個實作單使用）

> 欄位使用 snake_case，方便轉成 `const ECONOMY = { … }`。**本單不修改 app.js**，僅作規格交付。

### 8.1 任務分值

```js
const TASK_BASE_XP = {
  required: 25,
  optional: 15,
  skill_complete: 25, // finalizeSkillMission 觸發
};
```

## 9. 家長可調 vs 不可調項目清單

### 9.1 家長可調（在「家長端 → 獎勵設定／安全私隱」內）

| 項目 | 預設值 | 可調範圍 | 備註 |
|---|---|---|---|
| 每日 XP 上限 | 60 XP | 30–120 XP | 影響整體通脹速度 |
| 每日開包上限 | 2 包 | 1–4 包 | 影響收藏節奏 |
| 技能卡輪轉 | 開啟 | 開／關 | 關閉後由裝飾補位 |
| 本週主打區域 | 晨光碼頭、平靜海灣 | 1–2 個區域 | 影響卡包輪轉排序 |
| 通知推播 | 關 | 開／關 | 必須家長明確開啟 |
| 個別任務 XP 覆寫 | 0 | 5–100 XP | 對齊 `task.xp` |
| 任務難度標記 | 入門 | 入門／挑戰／勇敢一試 | 影響加成倍率 |
| 兌換獎勵清單 | 預設 2 條 | 任意新增 | 對齊 `state.rewards` |

### 9.2 不可調（硬編碼於規格）

- 卡包價格 3 顆探索星。
- 每包張數 1 張。
- 連續加成上限 ×1.35。
- 重複卡 XP 永遠為 0。
- 失敗不扣任何資源。
- 中斷連續不影響其他獎勵。
- 稀有度只影響外觀，不影響中獎率。
- 能力卡只能由技能任務解鎖，不會因開包次數自動升級。
- 每日裝置停留時間不設限制（家長觀察用）。
